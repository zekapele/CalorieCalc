# Розширені можливості проєкту

## Додаткові бібліотеки

### SQLite3
- **Призначення**: Зберігання даних у реляційній базі даних
- **Клас**: `SqliteSaveStrategy`
- **Використання**: Автоматично підключається, якщо SQLite3 знайдено системою
- **Переваги**: Швидкий пошук, SQL-запити, інтеграція з іншими системами

### Qt Network
- **Призначення**: REST API сервер для інтеграції
- **Клас**: `RestServer`
- **Збірка**: `cmake -DBUILD_API_SERVER=ON ..`
- **Ендпоінти**:
  - `GET /api/diary/{profile}/{date}` - отримати щоденник за датою
  - `GET /api/foods` - отримати всі продукти

## Розширені unit тести

### Нові тестові файли:
- `test_savedmeal.cpp` - тести для класу SavedMeal
- `test_statistics.cpp` - тести для StatisticsCalculator
- `test_savestrategies.cpp` - тести для всіх стратегій збереження (JSON, TXT, XML)

### Запуск тестів:
```bash
cmake -DBUILD_TESTS=ON ..
make
ctest
```

## Тести для бенчмарків

### Файл: `benchmarks/benchmark_tests.cpp`
- Валідація продуктивності пошуку в базі продуктів
- Валідація швидкості додавання прийомів їжі
- Валідація розрахунку калорій

### Запуск:
```bash
cmake -DBUILD_BENCHMARKS=ON ..
make
./CalorieCalcBenchmarkTests
```

## Додаткові сховища даних

### XML формат
- **Клас**: `XmlSaveStrategy`
- **Розширення**: `.xml`
- **Сумісність**: Стандартний XML, сумісний з іншими системами

### SQLite база даних
- **Клас**: `SqliteSaveStrategy`
- **Розширення**: `.db`
- **Структура**: Таблиці `diaries` та `meals`
- **SQL-запити**: Можна виконувати власні запити для аналізу

## Інтеграція з іншими додатками

### REST API сервер
- **Порт**: 8080 (за замовчуванням)
- **Протокол**: HTTP/1.1
- **Формат відповідей**: JSON
- **CORS**: Підтримується для веб-додатків

### Приклад використання:
```bash
# Запуск сервера
./CalorieCalcAPI 8080

# Отримати щоденник
curl http://localhost:8080/api/diary/guest/2025-01-15

# Отримати всі продукти
curl http://localhost:8080/api/foods
```

## Імпорт/експорт даних

### Підтримувані формати:
1. **JSON** - стандартний формат проєкту
2. **TXT** - текстовий формат для читання
3. **XML** - для сумісності з іншими системами
4. **CSV** - для Excel та інших табличних редакторів (GUI)
5. **PDF** - для друку та звітів (GUI)
6. **SQLite** - для програмного доступу

### Крос-форматна сумісність:
Дані можна зберігати в одному форматі та завантажувати з іншого.

## Покращене логування

### Нові можливості:
- **Ротація файлів**: Автоматичне створення нових файлів при досягненні ліміту
- **Мінімальний рівень**: Фільтрація повідомлень за рівнем важливості
- **Налаштування розміру**: Максимальний розмір файлу та кількість файлів

### Використання:
```cpp
Logger::getInstance().init("app.log", 10 * 1024 * 1024, 5); // 10MB, 5 файлів
Logger::getInstance().setMinLevel(Logger::Level::INFO); // Тільки INFO та вище
```

### Рівні логування:
- `DEBUG` - детальна інформація для розробки
- `INFO` - загальна інформація
- `WARNING` - попередження
- `ERROR` - помилки

## Збірка з усіма можливостями

```bash
# З тестами та бенчмарками
cmake -DBUILD_TESTS=ON -DBUILD_BENCHMARKS=ON -DBUILD_API_SERVER=ON ..

# З GUI
cmake -DBUILD_GUI=ON ..

# Все разом
cmake -DBUILD_TESTS=ON -DBUILD_BENCHMARKS=ON -DBUILD_API_SERVER=ON -DBUILD_GUI=ON ..
```
